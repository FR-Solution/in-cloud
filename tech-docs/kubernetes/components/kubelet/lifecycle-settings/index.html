5.2.1.3.4. Kubelet

5.2.1.3.4.1. Этапы установки компонента​

  • Создание рабочих директорий.
  • Настройка Systemd Unit (сервис).
  • Настройка компонента через Конфигурационные файлы существуют.
  • Запуск компонента и добавление сервиса в автозапуск.

5.2.1.3.4.2. Рабочие директории компонента​

mkdir -p /usr/lib/systemd/system/kubelet.service.d
mkdir -p /var/lib/kubelet/

5.2.1.3.4.3. Настройка Systemd Unit компонента​

Переменные окружния

export BASE_K8S_PATH="/etc/kubernetes"
export BASE_KUBELET_PATH="/var/lib/kubelet"
export CRI_ENDPOINT="/var/run/containerd/containerd.sock"
export BASE_DOCKER_REGISTRY="registry.k8s.io"
export PAUSED_IMAGE="pause:3.9"
export KUBELET_SERVER_PORT=10250
export KUBELET_HEALTHZ_PORT=10248

Базовая конфигурация systemd unit

cat <<EOF > /usr/lib/systemd/system/kubelet.service
[Unit]
Description=kubelet: The Kubernetes Node Agent
Documentation=https://kubernetes.io/docs/
Wants=network-online.target
After=network-online.target

[Service]
ExecStart=/usr/bin/kubelet
Restart=always
StartLimitInterval=0
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

Расширение конфигурации systemd unit

cat <<EOF > /usr/lib/systemd/system/kubelet.service.d/10-kubeadm.conf
# Note: This dropin only works with kubeadm and kubelet v1.11+
[Service]
Environment="KUBELET_KUBECONFIG_ARGS=--bootstrap-kubeconfig=${BASE_K8S_PATH}/bootstrap-kubelet.conf --kubeconfig=${BASE_K8S_PATH}/kubelet.conf"
Environment="KUBELET_CONFIG_ARGS=--config=${BASE_KUBELET_PATH}/config.yaml"
# This is a file that "kubeadm init" and "kubeadm join" generates at runtime, populating the KUBELET_KUBEADM_ARGS variable dynamically
EnvironmentFile=-${BASE_KUBELET_PATH}/kubeadm-flags.env
# This is a file that the user can use for overrides of the kubelet args as a last resort. Preferably, the user should use
# the .NodeRegistration.KubeletExtraArgs object in the configuration files instead. KUBELET_EXTRA_ARGS should be sourced from this file.
EnvironmentFile=-/etc/default/kubelet
ExecStart=
ExecStart=/usr/local/bin/kubelet \$KUBELET_KUBECONFIG_ARGS \$KUBELET_CONFIG_ARGS \$KUBELET_KUBEADM_ARG \$KUBELET_EXTRA_ARGS
EOF
cat <<EOF > ${BASE_KUBELET_PATH}/kubeadm-flags.env
KUBELET_KUBEADM_ARGS="
--container-runtime-endpoint=unix://${CRI_ENDPOINT} --pod-infra-container-image=${BASE_DOCKER_REGISTRY}/${PAUSED_IMAGE}
"
EOF
cat <<EOF > /etc/default/kubelet
KUBELET_EXTRA_ARGS=""
EOF

5.2.1.3.4.4. Старт Systemd Unit​

systemctl enable kubelet
примечание
Created symlink /etc/systemd/system/multi-user.target.wants/kubelet.service → /usr/lib/systemd/system/kubelet.service.

5.2.1.3.4.5. Конфигурация компонента​

warning

Просьба обратить внимание, что данный конфигурационный файл нужно применять только, если установка производится не через Kubeadm, в противном случае дубль конфигурации уже находится в kubeadm config

Kubelet config

Переменные окружния

export BASE_K8S_PATH="/etc/kubernetes"
export BASE_KUBELET_PATH="/var/lib/kubelet"
export KUBERNETES_CA_CRT_PATH="${BASE_K8S_PATH}/pki/ca.crt"
export SERVICE_DNS="29.64.0.10"
export CLUSTER_NAME="example"
export BASE_DOMAIN="example.com"
export CLUSTER_DOMAIN="${CLUSTER_NAME}.${BASE_DOMAIN}"
export KUBELET_CURRENT_SERVER_CRT_PATH="${BASE_KUBELET_PATH}/pki/kubelet-server-current.pem"
export KUBELET_HEALTHZ_PORT=10248
