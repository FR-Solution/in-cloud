5.2.3.2. Сертификаты

Сертификаты (Certificates) — это цифровые документы, которые подтверждают подлинность и надёжность участников взаимодействия в системе. В Kubernetes сертификаты используются для установления защищённых соединений между компонентами кластера, узлами и клиентами. Они основаны на инфраструктуре открытых ключей (PKI) и обеспечивают аутентификацию, авторизацию и шифрование данных. Каждый сертификат связан с закрытым ключом и содержит информацию о его владельце, сроке действия и удостоверяющем центре, выпустившем сертификат.

5.2.3.2.1. Kubelet client > K8S-API certificates​

5.2.3.2.1. Kubelet Client > Kubernetes API
  • HardWay
  • Kubeadm

5.2.3.2.1.1. Переменные окружения PKI​

export CLUSTER_NAME="example"
export BASE_DOMAIN="example.com"
export CLUSTER_DOMAIN="${CLUSTER_NAME}.${BASE_DOMAIN}"
export FULL_HOST_NAME="${HOST_NAME}.${CLUSTER_DOMAIN}"
export BASE_K8S_PATH="/etc/kubernetes"
export BASE_KUBELET_PATH="/var/lib/kubelet"
export KUBELET_CLIENT_CN=""
export KUBELET_CLIENT_CRT_CONF="${BASE_K8S_PATH}/openssl/kubelet-client.conf"
export KUBELET_CLIENT_CRT_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client.pem"
export KUBELET_CLIENT_CSR_PATH="${BASE_K8S_PATH}/openssl/csr/kubelet-client.csr"
export KUBELET_CLIENT_KEY_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client-key.pem"
export KUBELET_CLIENT_KEY_SIZE=2048
export KUBERNETES_CA_CRT_PATH="${BASE_K8S_PATH}/pki/ca.crt"
export KUBERNETES_CA_KEY_PATH="${BASE_K8S_PATH}/pki/ca.key"
export KUBELET_NOW_CLIENT_CRT_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client-$(date '+%Y-%m-%d-%H-%M-%S').pem"
export KUBELET_CURRENT_CLIENT_CRT_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client-current.pem"

5.2.3.2.2. Рабочая директория для PKI​

mkdir -p ${BASE_K8S_PATH}/pki
mkdir -p ${BASE_K8S_PATH}/openssl/csr
mkdir -p ${BASE_KUBELET_PATH}/pki

5.2.3.2.1.3. Конфигурация PKI​

cat <<EOF > ${KUBELET_CLIENT_CRT_CONF}
[ req ]
default_bits        = ${KUBELET_CLIENT_KEY_SIZE}
prompt              = no
default_md          = sha256
distinguished_name  = dn

[ dn ]
CN = system:node:${FULL_HOST_NAME}
O  = system:nodes

[ v3_ext ]
authorityKeyIdentifier=keyid,issuer:always
basicConstraints=CA:FALSE
keyUsage=keyEncipherment,dataEncipherment
extendedKeyUsage=clientAuth
EOF

5.2.3.2.1.4. Генерация приватного ключа​

openssl genrsa \
    -out "${KUBELET_CLIENT_KEY_PATH}" ${KUBELET_CLIENT_KEY_SIZE}

5.2.3.2.1.5. Генерация CSR​

openssl req \
    -new \
    -key "${KUBELET_CLIENT_KEY_PATH}" \
    -out "${KUBELET_CLIENT_CSR_PATH}" \
    -config "${KUBELET_CLIENT_CRT_CONF}"

5.2.3.2.1.6. Подпись CSR​

openssl x509 \
    -req \
    -days 365 \
    -sha256 \
    -outform PEM \
    -CA "${KUBERNETES_CA_CRT_PATH}" \
    -CAkey "${KUBERNETES_CA_KEY_PATH}" \
    -CAcreateserial \
    -in "${KUBELET_CLIENT_CSR_PATH}" \
    -out "${KUBELET_CLIENT_CRT_PATH}" \
    -extensions v3_ext \
    -extfile "${KUBELET_CLIENT_CRT_CONF}"
cat "${KUBELET_CLIENT_CRT_PATH}" "${KUBELET_CLIENT_KEY_PATH}" >> "${KUBELET_NOW_CLIENT_CRT_PATH}"
ln -s "${KUBELET_NOW_CLIENT_CRT_PATH}" "${KUBELET_CURRENT_CLIENT_CRT_PATH}"

5.2.3.2.1.7. Kubeconfig​

export CLUSTER_NAME="example"
export BASE_DOMAIN="example.com"
export CLUSTER_DOMAIN="${CLUSTER_NAME}.${BASE_DOMAIN}"
export FULL_HOST_NAME="${HOST_NAME}.${CLUSTER_DOMAIN}"
export BASE_K8S_PATH="/etc/kubernetes"
export KUBELET_CLIENT_CRT_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client.pem"
export KUBELET_CLIENT_KEY_PATH="${BASE_KUBELET_PATH}/pki/kubelet-client-key.pem"
export KUBERNETES_CA_CRT_PATH="${BASE_K8S_PATH}/pki/ca.crt"

Инструкция создания Kubeconfig

kubectl config set-cluster kubernetes \
    --certificate-authority="${KUBERNETES_CA_CRT_PATH}" \
    --embed-certs=true \
    --server=https://127.0.0.1:6443 \
    --kubeconfig=${BASE_K8S_PATH}/kubelet.conf

kubectl config set-credentials system:node:${FULL_HOST_NAME} \
    --client-certificate=${KUBELET_CLIENT_CRT_PATH} \
    --client-key=${KUBELET_CLIENT_KEY_PATH} \
    --embed-certs=true \
    --kubeconfig=${BASE_K8S_PATH}/kubelet.conf

kubectl config set-context default \
    --cluster=kubernetes \
    --user=system:node:${FULL_HOST_NAME} \
    --kubeconfig=${BASE_K8S_PATH}/kubelet.conf

kubectl config use-context default \
    --kubeconfig=${BASE_K8S_PATH}/kubelet.conf

Генерация Kubelet сертификатов​

kubeadm init phase kubeconfig kubelet \
    --config=${BASE_K8S_PATH}/kubeadm.conf
kubeadm init phase kubelet-finalize experimental-cert-rotation \
    --config=${BASE_K8S_PATH}/kubeadm.conf
примечание

После выполнения команд мы получаем следующий вывод.

#### Генерация Kubelet сертификатов
[kubeconfig] Writing "kubelet.conf" kubeconfig file
