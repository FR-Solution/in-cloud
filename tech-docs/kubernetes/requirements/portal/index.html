2.7. Портал

2.7.1. Общие требования к интерфейсу​

№Требование
1Портал должен быть реализован через графический web-интерфейс.
2Портал должен быть доступен к использованию в различных браузерах (Google Chrome, Edge, Firefox).
3Портал должен иметь адаптивный интерфейс под различные размеры экранов для работы на различных устройствах.
4Портал должен поддерживать мультиязычный интерфейс.
5Портал должен реализовывать функционал уведомления пользователей в интерфейсе платформы.
6Портал должен поддерживать функционал мультитенантности и обеспечивать управление видимостью объектов на основе RBAC.
7Портал должен обеспечивать интерфейс доступа к управлению выделенными ресурсами, в том числе консолям виртуальных машин.
8Портал должен иметь возможность настройки стиля под корпоративный шаблон заказчика.

2.7.2.2. Общие требования к каталогу сервисов​

№Требование
1Каталог сервисов должен поддерживать создание услуг на основе подготовленных образов ВМ.
2Каталог сервисов должен поддерживать создание услуг по предоставлению ПО, развернутого в контейнерах k8s.
3Каталог сервисов должен поддерживать создание услуг на основе образов контейнеров k8s или ВМ с возможностью последующего добавления и настройки необходимого ПО с помощью технологий автоматизации (Ansible, Terraform).
4Каталог сервисов должен поддерживать возможность предоставления SaaS-приложений.
5Каталог сервисов должен поддерживать публикацию услуг/перенос услуг в архив.
6Каталог сервисов при заказе услуги должен поддерживать управление версиями ПО, входящих в услугу.
7Платформа должна поддерживать заказ услуг из каталога сервисов.
8Платформа должна поддерживать возможность интеграции с системами workflow для реализации процесса согласования услуги внутри платформы.
9Платформа должна поддерживать возможность интеграции с системами workflow для реализации процесса развертывания услуги внутри платформы.
10Каталог сервисов должен поддерживать функционал поиска и фильтрации услуг по атрибутам/тегам.
11Каталог сервисов должен поддерживать функционал категоризации услуг.

2.7.3. Требования к разделу администрирования​

№Требование
1Портал администрирования должен предоставлять интерфейс для отображения информации об объектах облачной инфраструктуры.
2Портал администрирования должен предоставлять интерфейс для отображения отчетов об использовании объектов облачной инфраструктуры: текущей и исторической утилизации.
3Портал администрирования должен предоставлять интерфейс для управления ролями и доступом к платформе/проектам (в том числе на основе RBAC).
4Портал администрирования должен предоставлять интерфейс для управления жизненным циклом внутренних учетных записей пользователей облака.
5Портал администрирования должен предоставлять интерфейс для управления проектами (создание, изменение, архивирование).
6Портал администрирования должен предоставлять интерфейс для управления ресурсными квотами.
7Портал администрирования должен предоставлять интерфейс для управления тарификацией для проектов.
8Портал администрирования должен предоставлять интерфейс для отображения информации о проекте (ресурсы, квоты, тарифы, баланс и пр.).
9Портал администрирования должен предоставлять интерфейс для отображения статусов услуг из каталога.
10Портал администрирования должен предоставлять интерфейс для управления каталогом сервисов.
11Портал администрирования должен поддерживать функционал управления шаблонами и образами ОС, IaaS, PaaS, SaaS сервисами и пр.

2.7.4. Требования к интерфейсу и обучению​

№Требование
1Поддерживает локализацию пользовательского интерфейса (русский, казахский, английский).
2Наличие демонстрационного стенда для обучения.
3Имеются обучающие курсы/видео по работе с продуктом.
4Ожидаемая продолжительность изучения системы пользователем не превышает 10 часов.
5Имеется пользовательский интерфейс для администрирования.
6Имеется функционал по проверке корректности вводимых пользователем данных (валидация).
7Возможно настроить отображение пользовательского интерфейса в соответствии с дизайн-системой банка.
8Имеется встроенная справочная система/система подсказок, облегчающая работу начинающего пользователя.
9Элементы интерфейса, предусматривающие взаимодействие с пользователем, должны визуально выделяться относительно неактивных.
10Система позволяет настраивать всплывающие подсказки, возникающие при наведении фокуса на элемент интерфейса.
11Система должна корректно обрабатывать конфликт одновременных изменений одного объекта несколькими пользователями.
12Все произведённые пользователем настройки должны сохраняться индивидуально для каждого авторизованного пользователя.
13Административные настройки пользователей и ролей должны быть централизованы (собраны в едином интерфейсе системы).
14Схожие по функциональности элементы интерфейса, предусматривающие взаимодействие с пользователем, должны выглядеть единообразно по всей Системе.
