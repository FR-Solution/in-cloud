Ресурс Project

Project — представляет собой введенную абстракцию, созданную для обеспечения команды отдельным пространством имен, предназначенным для управления и развертывания необходимых инфраструктурных ресурсов. Основная цель проекта — предоставить клиенту выделенную область для управления информационными системами, где можно организовать доступ разработчиков и администраторов, гибко контролировать выделенные ресурсы и квоты, отслеживать состояние развернутых систем и интегрироваться с инструментами DevOps для эффективного управления инфраструктурой.

Поля ресурса​

  • Описание

  • spec - контент, определяющий проект.
  • spec.businessName - полное бизнес-имя проекта.
  • spec.description - краткое описание проекта.
  • spec.prefix - короткий идентификатор проекта.
  • spec.quotas - квоты на ресурсы проекта, разделенные на лимиты и запросы для CPU, RAM и DISK.
  • spec.quotas.k8s - квоты на ресурсы проекта, разделенные на лимиты и запросы для CPU, RAM и DISK для K8S.
  • spec.quotas.k8s.limits - лимиты для CPU, RAM в проекте.
  • spec.quotas.k8s.limits.cpu - лимит по CPU.
  • spec.quotas.k8s.limits.memory - лимит по RAM.
  • spec.quotas.k8s.requests - запросы для CPU, RAM в проекте.
  • spec.quotas.k8s.requests.cpu - запросы на CPU.
  • spec.quotas.k8s.requests.memory - запросы на RAM.
  • spec.quotas.k8s.requests.storage - запросы на DISK.
  • state - тип проекта (DEPRECATED).
  • status - структура, определяющая текущее состояние проекта.
  • status.conditions[] - массив, содержащий условия текущего состояния проекта.
  • status.conditions[].lastTransitionTime - время последнего изменения состояния.
  • status.conditions[].message - сообщение с деталями о переходе состояния.
  • status.conditions[].observedGeneration - наблюдаемая версия объекта.
  • status.conditions[].reason - идентификатор причины перехода состояния.
  • status.conditions[].status - итоговое состояние.
  • status.conditions[].type - тип условия.
  • status.status - текущее состояние проекта.

  • Валидация создания ресурса

    • metadata.labels:

      • Ключи и значения меток должны быть строками.
      • Максимальная длина ключей и значений: 63 символа.
      • Допускаются только буквы, цифры, дефисы, точки и символ _.
      • Не допускается дублирование ключей.

    • metadata.name:

      • Имя должно быть строкой.
      • Максимальная длина имени: 253 символа.
      • Допускаются только буквы, цифры (0-9), дефисы (-), и символ _.
      • Имя не должно начинаться или заканчиваться дефисом (-) или _.

    • spec.businessName:

      • Бизнес-имя должно быть строкой.
      • Максимальная длина: 255 символов.
      • Допускаются буквы, пробелы и базовые знаки пунктуации (точка, запятая, тире).

    • spec.prefix:

      • Префикс должен быть строкой.
      • Максимальная длина: 7 символов.
      • Допускаются только буквы и цифры.
      • Не должно начинаться или заканчиваться дефисом.

    • spec.state:

      • Поле принимает ограниченный набор значений (например: active, inactive, pending, archived).

    • spec.description:

      • Значение должно быть строкой.
      • Максимальная длина: 255 символов.
      • Поле может быть пустым.

    • spec.quotas:

      • Квоты должны быть корректно преобразуемыми в "hard limits".
      • Значения квот числовые и неотрицательные.
      • Поле должно содержать определенные ресурсы (CPU, Memory, Storage).

    • spec.quotas restriction:

      • Значения ресурсов в квотах не должны превышать установленные ограничения.
  • Валидация обновления ресурса

    • metadata.labels:

      • Ключи и значения меток должны быть строками.
      • Максимальная длина ключей и значений: 63 символа.
      • Допускаются только буквы, цифры, дефисы, точки и символ _.
      • Не допускается дублирование ключей.

    • metadata.name:

      • Имя должно быть строкой.
      • Максимальная длина имени: 253 символа.
      • Допускаются только буквы, цифры (0-9), дефисы (-), и символ _.
      • Имя не должно начинаться или заканчиваться дефисом (-) или _.

    • spec.businessName:

      • Бизнес-имя должно быть строкой.
      • Максимальная длина: 255 символов.
      • Допускаются буквы, пробелы и базовые знаки пунктуации (точка, запятая, тире).

    • spec.prefix:

      • Префикс должен быть строкой.
      • Максимальная длина: 7 символов.
      • Допускаются только буквы и цифры.
      • Не должно начинаться или заканчиваться дефисом.

    • spec.state:

      • Поле принимает ограниченный набор значений (например: active, inactive, pending, archived).

    • spec.description:

      • Значение должно быть строкой.
      • Максимальная длина: 255 символов.
      • Поле может быть пустым.

    • spec.quotas:

      • Квоты обновляемого проекта не должны превышать суммы ресурсов всех инстансов проекта.
      • Значения квот (CPU, Memory, Storage, и т.д.) должны быть числовыми и допустимыми.
      • Некорректные или отрицательные значения запрещены.
      • Квоты всех инстансов должны быть суммированы и не должны превышать новые значения квот.
      • Ошибки преобразования квот инстансов в "hard limits" недопустимы.
      • Проверяется соответствие для каждого ресурса (CPU, Memory, и т.д.).
  • Валидация удаления ресурса
    • Отсутствует
