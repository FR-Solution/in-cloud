# Тюнинг

## Размер БД ETCD


1.  Максимальный размер запроса (Request size limit)
    ETCD спроектировано для обработки небольших пар ключ/значение типичных для метадаты. Запросы большого размера будут работать, но могут увеличить задержку других запросов. По умолчанию, максимальный размер любого запроса - 1,5 MiB. Это ограничение конфигурируемое при помощи флага `--max-request-bytes` для ETCD сервера.

2.  Максимальный размер хранилища (Storage size limit)
    По умолчанию максимальный размер данных на диске используемых БД ETCD равен 2GB. Это значение настраивается при помощи флага `--quota-backend-bytes`. 8GB предлагаемый максимальный размер данного хранилища и при указании значения больше 8GB в логах сервиса будут соответствующие предупреждения.
