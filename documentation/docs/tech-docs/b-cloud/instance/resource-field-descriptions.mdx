<ul>
    <li>`spec.spi`                                  - доступность персональной информации.</li>
    <li>`spec.chatid`                               - идентификатор чата в telegram для отпроавки уведомлений.</li>
    <li>`spec.description`                          - краткое описание инстанса.</li>
    <li>`spec.quotas`                               - квоты на ресурсы проекта, разделенные на лимиты и запросы для CPU, RAM и DISK.</li>
    <li>`spec.quotas.limits`                        - лимиты для CPU, RAM в проекте.</li>
    <li>`spec.quotas.limits.cpu`                    - лимит по CPU.</li>
    <li>`spec.quotas.limits.memory`                 - лимит по RAM.</li>
    <li>`spec.quotas.requests`                      - запросы для CPU, RAM в проекте.</li>
    <li>`spec.quotas.requests.cpu`                  - запросы на CPU.</li>
    <li>`spec.quotas.requests.memory`               - запросы на RAM.</li>
    <li>`spec.quotas.requests.storage`              - запросы на DISK.</li>
    <li>`status`                                    - структура, определяющая текущее состояние проекта.</li>
    <li>`status.conditions[]`                       - массив, содержащий условия текущего состояния проекта.</li>
    <li>`status.conditions[].lastTransitionTime`    - время последнего изменения состояния.</li>
    <li>`status.conditions[].message`               - сообщение с деталями о переходе состояния.</li>
    <li>`status.conditions[].observedGeneration`    - наблюдаемая версия объекта.</li>
    <li>`status.conditions[].reason`                - идентификатор причины перехода состояния.</li>
    <li>`status.conditions[].status`                - итоговое состояние.</li>
    <li>`status.conditions[].type`                  - тип условия.</li>
    <li>`status.status`                             - текущее состояние проекта.</li>
</ul>
